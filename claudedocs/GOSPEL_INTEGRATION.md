# TaskBridge 福音伝道統合戦略

**文書分類**: 機密 - 内部戦略資料
**作成日**: 2025-01-11
**対象読者**: 日本側計画チーム、派遣宣教師候補者
**目的**: TaskBridgeプラットフォームを通じた福音伝道活動の包括的戦略と実装計画

---

## 📋 エグゼクティブサマリー

TaskBridgeは、アフリカ・アジアのワーカーに経済的機会を提供するマイクロタスクプラットフォームです。この戦略文書は、プラットフォームのビジネス中立性を保ちながら、ワーカーコミュニティに福音を伝えるための包括的アプローチを提示します。

### 核心的アプローチ

**二層アーキテクチャ**:
- **Layer 1（プラットフォーム内）**: 完全に世俗的な職業教育プログラム
- **Layer 2（プラットフォーム外）**: 自然な関係構築を通じた福音伝道

### 重点地域

**Phase 1**: ベトナム（ホーチミン市）
- キリスト教徒比率: 約8%
- 政府による宗教活動規制あり
- 慎重な二層分離アプローチが必須

**Phase 2以降**: ガーナ、ナイジェリア（既存キリスト教徒比率が高い）

### 成功の鍵

1. ✅ **本物の価値提供**: ワーカーの収入を実際に25-30%向上させる
2. ✅ **完全な任意参加**: 強制感ゼロ、オプトアウト容易
3. ✅ **法的コンプライアンス**: 教育活動と信仰活動の完全分離
4. ✅ **測定可能性**: 客観的指標で御心に沿っているか評価

### Layer 2: プラットフォーム外（オフライン活動）

#### 現地拠点

**表向きの名称**: "TaskBridge Skills Center - Ho Chi Minh City"

**法的登録**: 教育・職業訓練サービス提供機関として正式登録

**運営体制**:
- 宣教師（肩書き: "Training Coordinator"）
- 必要に応じてベトナム語通訳ボランティア

**物理的拠点**:
- Phase 1: コワーキングスペースの会議室レンタル（$60-180/週）
- Phase 2: 小規模オフィスの長期レンタル（$300-500/月、10-15名収容）

#### 活動フローと時間軸

**Month 1-2: 純粋な職業訓練フェーズ**

```
【公式活動】
• 実際のスキル研修（英語、デジタルツール、ビジネスマナー）
• 小グループ形式（3-5人）
• 週2-3回のセッション
• すべて記録あり、政府査察に耐えられる内容

【宣教師の姿勢】
• 100%職業訓練に集中
• 信仰の話題は一切出さない
• プロフェッショナルな教育者として振る舞う
```

**Month 3-6: 関係構築フェーズ**

```
【自然な対話の発生】
• 休憩時間・食事時の雑談
• ワーカーの生活課題の傾聴
• 実践的サポート（キャリア相談、履歴書添削）
• 信頼関係の構築

【宣教師の姿勢】
• ワーカー主導の対話を待つ
• 聞き役に徹する
• 個人的な悩みに共感的に応答
• 信仰の話題は出さない（この段階では）
```

**Month 6+: 個別対応フェーズ**

```
【ワーカーの関心に応じて】
• 関心のある人: 個別に人生観・価値観の対話開始
• 関心のない人: 純粋なスキルサポートのみ継続

【任意のコミュニティイベント】
• 月1回の "Networking Dinner"
• 文化交流会、ゲームナイトなど
• 完全に任意参加
• 自然な形で信仰の話題が出てくる環境

【重要原則】
• 強制は一切しない
• スキルトレーニングは常に本物（質を妥協しない）
• 信仰に興味ない人も引き続き歓迎
• プラットフォームとの関係は「教育パートナー」のみ
```

#### 時間・場所の完全分離（法的コンプライアンス）

```
【公式な職業訓練】
時間: 月-金 9AM-6PM
場所: 登録済みトレーニングセンター
内容: 純粋な教育活動のみ
記録: すべて記録、政府提出可能

【私的なコミュニティ活動】
時間: 土日、平日夜間
場所: 宣教師の自宅、カフェ、または別の場所
内容: 任意の集まり、信仰関連の対話可能
記録: 私的な集まりとして記録なし
```

## 🇻🇳 ベトナム特化戦略

### なぜベトナムを最初の重点地域とするか

**戦略的理由**:
1. キリスト教徒比率が低い（約8%）→ 伝道の必要性が高い
2. 政府規制がある → 慎重なアプローチの実践訓練になる
3. TaskBridgeのワーカー増加が見込まれる地域
4. ホーチミン市は外資企業・NGO活動が活発（教育機関登録が比較的容易）

**文化的考慮**:
- 教育への高い価値観 → 職業訓練への参加動機が強い
- 集団主義的文化 → グループ学習が効果的
- 実用主義 → 「収入が増える」が最強の動機
- 仏教・儒教的背景 → 対話的アプローチが適合

## 🎓 職業教育プログラムの詳細

### 3層カリキュラム構造

#### Level 1: Foundation Program（基礎プログラム）

**期間**: 6週間、週3回（火・木・土）、各2時間
**対象**: すべての新規参加ワーカー
**形式**: 小グループ（3-5名）
**費用**: 完全無料

##### Week 1-2: English for Work（仕事の英語）

```
【内容】
• TaskBridge特化の実践英語:
  - タスク説明の読解
  - 質問の仕方（How to ask clarification questions）
  - 成果物の説明文の書き方
  - 顧客とのプロフェッショナルなコミュニケーション

• 実践演習:
  - 実際のタスク説明を使った読解練習
  - ロールプレイ: 顧客とのやりとり
  - 自分の成果物説明文を添削

【教材】
• ビデオレッスン（Loom、10-15分）
• ワークシート（Google Docs）
• 実際のTaskBridgeタスクサンプル

【ワーカーへの価値】
→ タスク理解度向上 → ミス減少 → 評価アップ
```

##### Week 3-4: Digital Tools Mastery（デジタルツール習得）

```
【内容】
• TaskBridge業務に直結するツール:
  - Google Workspace（Docs, Sheets, Forms）

  - ファイル管理とクラウドストレージ
  - スマートフォンの活用術（低スペック端末でも）

• 実践演習:
  - 各ワーカーが実際のタスクで使うツールをセットアップ
  - 効率的なワークフロー構築
  - トラブルシューティング

【教材】
• デモビデオ（画面録画）

【ワーカーへの価値】

```
【内容】

【教材】

【ワーカーへの価値】

- 出席率 > 80%

### プログラムの費用構造

#### 成果連動モデル（推奨）

```
Level 1（基礎）:
• ワーカー費用: 無料
• 支援側負担: 全額

Level 2（応用）:
• ワーカー費用: 無料（前払いなし）
• 成果連動寄付: 修了後6ヶ月間、収入増加分の10%を任意寄付
• 支援側負担: 初期費用全額

Level 3（リーダーシップ）:
• ワーカー費用: 月$10の会費
• 支援側負担: 一部（不足分）

【例】
ワーカーAさん（Level 2修了）:
- プログラム前の月収: $100
- プログラム後の月収: $150
- 増加分: $50/月
- 任意寄付（10%）: $5/月 × 6ヶ月 = $30
- ワーカーの実質利益: $50 × 6 = $300 - $30 = $270

【メリット】
✅ ワーカーは初期費用ゼロでスタート
✅ プログラムの価値を実感してから支払い
✅ 収入向上した人のみが貢献
✅ 持続可能な運営モデル（支援依存度を徐々に減少）
```

## 💻 技術実装仕様

### データベーススキーマ

```sql
-- 教育プログラムテーブル
CREATE TABLE education_programs (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  name TEXT NOT NULL,
  description TEXT,
  region TEXT NOT NULL, -- 'vietnam', 'ghana', 'nigeria', etc.
  level INTEGER, -- 1, 2, 3
  track TEXT, -- 'foundation', 'translation', 'ai', 'testing', 'leadership'
  partner_organization TEXT,
  curriculum JSONB, -- プログラム内容の詳細
  schedule JSONB, -- スケジュール情報
  status TEXT DEFAULT 'active', -- 'active', 'paused', 'closed'
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- プログラム参加希望テーブル
CREATE TABLE program_interests (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  worker_id UUID REFERENCES users(id) ON DELETE CASCADE,
  program_id UUID REFERENCES education_programs(id) ON DELETE CASCADE,
  interest_areas JSONB, -- ["digital_skills", "english", "business"]
  preferred_schedule TEXT,
  status TEXT DEFAULT 'pending', -- 'pending', 'contacted', 'enrolled', 'active', 'completed', 'withdrawn'
  contacted_at TIMESTAMPTZ,
  enrolled_at TIMESTAMPTZ,
  completed_at TIMESTAMPTZ,
  coordinator_notes TEXT, -- 宣教師用の記録欄（プライベート）
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(worker_id, program_id)
);

-- プログラム活動ログ（任意、分析用）
CREATE TABLE program_activities (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  worker_id UUID REFERENCES users(id) ON DELETE CASCADE,
  program_id UUID REFERENCES education_programs(id) ON DELETE CASCADE,
  activity_type TEXT, -- 'session_attended', 'milestone_completed', 'assessment_passed'
  activity_date DATE,
  notes TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- プログラム成果追跡（収入変化の測定）
CREATE TABLE program_outcomes (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  worker_id UUID REFERENCES users(id) ON DELETE CASCADE,
  program_id UUID REFERENCES education_programs(id) ON DELETE CASCADE,
  metric_type TEXT, -- 'task_quality_score', 'monthly_income', 'tier_status'
  value_before NUMERIC,
  value_after NUMERIC,
  measured_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### Row Level Security (RLS) ポリシー

```sql
-- ワーカーは自分の参加記録のみ閲覧可能
CREATE POLICY "Workers can view own program interests"
  ON program_interests FOR SELECT
  TO authenticated
  USING (auth.uid() = worker_id);

-- ワーカーは自分の参加希望を登録可能
CREATE POLICY "Workers can register program interest"
  ON program_interests FOR INSERT
  TO authenticated
  WITH CHECK (auth.uid() = worker_id);

-- ワーカーは自分の参加をキャンセル可能（withdraw）
CREATE POLICY "Workers can withdraw from programs"
  ON program_interests FOR UPDATE
  TO authenticated
  USING (auth.uid() = worker_id)
  WITH CHECK (
    auth.uid() = worker_id
    AND status IN ('pending', 'contacted', 'enrolled', 'active', 'withdrawn')
  );

-- 管理者（宣教師/コーディネーター）はすべての記録を閲覧・更新可能
CREATE POLICY "Admins can manage program interests"
  ON program_interests FOR ALL
  TO authenticated
  USING (
    EXISTS (
      SELECT 1 FROM users
      WHERE id = auth.uid() AND role IN ('admin', 'coordinator')
    )
  );

-- coordinator_notes は管理者のみ閲覧可能（ワーカーには見えない）
-- → SELECT時にcoordinator_notesを除外するか、別テーブルに分離する必要あり
```

## 🛡️ リスク管理とコンプライアンス

### 4つの主要リスクと軽減策

#### リスク1: ベトナム政府による宗教活動規制違反

**具体的リスク**:
- 教育機関として登録しているのに宗教活動を行っていると判断される
- 政府による査察または閉鎖命令
- 宣教師のビザ取り消し

**軽減策**:

```
【プラットフォーム上の完全分離】
✅ 宗教的用語を一切使用しない
✅ 全ての表現を「職業訓練」「スキル開発」に限定
✅ ワーカー登録フォームに宗教関連項目なし

【現地拠点の法的適合性】
✅ 教育機関として正式登録
✅ カリキュラムは実質的な職業訓練内容
✅ 公式トレーニング時間（月-金 9AM-6PM）は記録を完備
✅ 政府査察に耐えられる運営実態

【時間・場所の完全分離】
✅ 公式トレーニング: 登録施設内、記録あり
✅ 信仰関連の対話: 土日・夜間、別の場所、私的な集まり

【証言の整合性】
✅ ワーカーが「スキル研修に参加した」と証言できる実態
✅ 実際に収入が向上している証拠
✅ 強制や圧力が一切ない証拠（匿名フィードバック）
```

**モニタリング**:
- 月次: 現地の法規制変更をチェック
- 四半期: 法律専門家によるコンプライアンスレビュー
- 問題発生時: 即座に活動を一時停止し、評価

---

#### リスク2: プラットフォームのビジネス中立性喪失

**具体的リスク**:
- 企業顧客が「宗教色のあるプラットフォーム」と認識
- ワーカーが「特定の宗教団体に支配されている」と感じる
- 他の教育団体が参入しにくい閉鎖的な構造

**軽減策**:

```
【汎用フレームワーク設計】
✅ "Skills Development Hub"は汎用プラットフォーム
✅ 将来的に他の教育団体も同じ仕組みで参加可能
✅ 特定団体を優遇しない中立的な表示

【マーケティング表現】
✅ 企業顧客向け: "Worker skill development partnership"
✅ 「宗教」「ミッション」という言葉は使わない
✅ "We partner with local training organizations to upskill workers"

【透明性と選択肢】
✅ ワーカーに複数の教育オプションを提示（将来的に）
✅ 参加は完全任意、プラットフォーム利用に影響なし
✅ オプトアウトが簡単
```

**モニタリング**:
- 月次: 企業顧客のフィードバック収集
- 四半期: ワーカーNPS（Net Promoter Score）測定
- 年次: 独立した第三者による中立性評価

---

#### リスク3: ワーカーが強制や不利益を感じる

**具体的リスク**:
- 「参加しないとタスクが割り当てられない」と誤解
- 「信仰を強要されている」と感じて離脱
- 信仰に興味がないワーカーが疎外感を感じる

**軽減策**:

```
【UI上の明確な表示】
✅ "This is optional and does not affect your task opportunities"
✅ 参加/非参加に関わらず、タスクアクセスは平等
✅ いつでも簡単にオプトアウト可能

【定期的なフィードバック収集】
✅ プログラム参加者に匿名アンケート（月次）
  - 「圧力を感じたことはありますか？」
  - 「自由に辞められると感じますか？」
  - 「信仰について話すことを強制されましたか？」
✅ 問題の兆候があれば即座に対応

【非参加ワーカーの継続率モニタリング】
✅ 非参加ワーカーの継続率 = 参加ワーカーの継続率
✅ 差がある場合は、システムに問題があると判断

【宣教師のトレーニング】
✅ 「強制しない」「ワーカー主導」の原則を徹底
✅ 信仰に興味がない人も平等に扱う訓練
```

**早期警告指標**:
- 匿名フィードバックで「強制」「圧力」の言及: 即座に調査
- 参加者の突然の大量離脱（1ヶ月に3名以上）: 原因調査
- SNS/フォーラムでのネガティブ言及: 即座に対応

---

#### リスク4: 文化的摩擦・誤解

**具体的リスク**:
- ベトナムの文化・歴史を理解せずに西洋的価値観を押し付ける
- 仏教徒や無宗教者を軽視する態度
- 現地の社会規範に反する行動

**軽減策**:

```
【宣教師の徹底した文化理解トレーニング】
✅ 派遣前: 3-6ヶ月のベトナム文化・歴史研修
✅ ベトナム語の基礎習得（最低限の日常会話）
✅ 仏教・儒教的価値観の理解
✅ ベトナム戦争の歴史と現代への影響

【対話のペース: ワーカー主導】
✅ 宣教師は「聞き役」に徹する
✅ ワーカーが話したいペースで進める
✅ 信仰の話題はワーカーから出てくるまで待つ

【多様性の尊重】
✅ 仏教徒、無宗教者も引き続き歓迎
✅ スキルトレーニングは誰にでも平等に提供
✅ 「改宗」を目標とせず、「福音を知る機会」を提供

【現地アドバイザーの活用】
✅ ベトナム人クリスチャンをアドバイザーとして
✅ 文化的に不適切な行動をチェック
```

---

### 総合リスクスコア

四半期ごとに以下のスコアリングを実施:

```
🟢 GREEN ZONE (Risk Score: 0-2):
  - すべてのリスク指標がクリア
  - 継続・拡大OK

🟡 YELLOW ZONE (Risk Score: 3-5):
  - 一部リスク指標に黄色信号
  - 改善計画を策定し、3ヶ月以内に改善
  - 新規拡大は一時停止

🔴 RED ZONE (Risk Score: 6-10):
  - 重大なリスク指標に問題
  - プログラムを即座に一時停止
  - 根本原因分析と対策立案
  - 外部専門家によるレビュー
```

### 四半期評価フレームワーク

**3ヶ月ごとに以下の総合評価を実施**:

```
🟢 GREEN ZONE (継続・拡大OK):
  条件:
  ✅ すべてのリスク指標 = 0件
  ✅ 教育プログラムの実効性指標 > 目標の80%
  ✅ 参加者満足度 > 4.0/5.0
  ✅ 非参加ワーカー継続率 = 参加者と同等

  アクション:
  → 現状維持または拡大を検討
  → 次フェーズへの移行を検討

🟡 YELLOW ZONE (改善必要):
  条件:
  ⚠️ リスク指標はクリア（0件）
  ⚠️ 一部の実効性指標が目標未達
  ⚠️ 参加者満足度 3.5-4.0

  アクション:
  → 3ヶ月の改善期間を設定
  → 具体的な改善計画を策定
  → 新規拡大は一時停止
  → 次回評価で再判定

🔴 RED ZONE (一時停止):
  条件:
  🚨 リスク指標に問題発生（1件でも）
  または
  🚨 参加者満足度 < 3.5
  または
  🚨 非参加ワーカー継続率が参加者より10%以上低い

  アクション:
  → プログラムを即座に一時停止
  → 根本原因分析（5 Whys手法）
  → 外部専門家によるレビュー
  → 対策立案と承認
  → 再開の可否判断
```

### 測定の実施方法

#### 自動測定（プラットフォームデータ）

```sql
-- 参加者のタスク品質スコア変化
SELECT
  pi.worker_id,
  AVG(t.client_rating) FILTER (WHERE t.created_at < pi.enrolled_at) as score_before,
  AVG(t.client_rating) FILTER (WHERE t.created_at > pi.completed_at) as score_after
FROM program_interests pi
JOIN tasks t ON t.worker_id = pi.worker_id
WHERE pi.status = 'completed'
GROUP BY pi.worker_id;

-- 参加者の月収変化
-- 同様のクエリでタスク収益を集計

-- Pro Tier昇格率（参加者 vs 非参加者）
SELECT
  CASE WHEN pi.worker_id IS NOT NULL THEN 'participant' ELSE 'non-participant' END as group,
  COUNT(*) FILTER (WHERE u.tier = 'pro') * 100.0 / COUNT(*) as upgrade_rate
FROM users u
LEFT JOIN program_interests pi ON pi.worker_id = u.id
WHERE u.role = 'worker' AND u.region = 'vietnam'
GROUP BY group;
```
