## 📂 提出ファイル形式チェック

電子申請システムでは、以下のファイル形式が求められます:

| ファイル種類 | 形式 | 最大サイズ |
|------------|------|-----------|
| 申請書・計画書 | PDF | 10MB |
| 経費明細書 | Excel | 5MB |
| 見積書 | PDF | 各5MB |
| 決算書・確定申告書 | PDF | 10MB |
| 通帳コピー | PDF | 5MB |
| 履歴事項全部証明書 | PDF | 5MB |

- [ ] 全ファイルが上記形式・サイズに適合している
- [ ] ファイル名が分かりやすい(例: 「TaskBridge_様式5_補助金交付申請書.pdf」)
- [ ] PDFファイルはテキスト検索可能な状態(スキャン画像の場合はOCR処理済み)

## ⚠️ よくあるミス・注意点

### 1. 日付の誤り
- [ ] 事業開始日が交付決定日より前になっていないか
- [ ] 見積書の日付が申請日より前か
- [ ] 履歴事項全部証明書の発行日が3ヶ月以内か

### 2. 金額の誤り
- [ ] 補助対象経費合計と内訳の合計が一致しているか
- [ ] 補助金額の計算が正しいか(補助率2/3、賃金引上げ枠上限2,000,000円)
- [ ] 消費税の取り扱いが正しいか(税込価格で申請)

### 3. 必須項目の記入漏れ
- [ ] GビズIDプライムアカウントIDが記載されているか
- [ ] 申請者情報(住所・電話番号・メールアドレス)がすべて記載されているか
- [ ] 誓約事項にチェックが入っているか

### 4. 見積書の不備
- [ ] 見積書に発注者名(TaskBridge)が記載されているか
- [ ] 作業内容が具体的に記載されているか
- [ ] 単価・数量・合計が明記されているか
- [ ] 見積書の有効期限が切れていないか

### 5. 自己資金証明の不備
- [ ] 通帳コピーに口座名義が表示されているか
- [ ] 残高が自己負担額以上あるか
- [ ] 直近3ヶ月分の取引明細が含まれているか

## 💡 申請のポイント

### 1. 事業の独自性を明確に
- 既存のクラウドソーシングとの違いを明確に記載
- 4つのフォーカス領域の専門性を強調
- 自動化システムによるスケーラビリティをアピール

### 2. 具体的な数値目標を設定
- 定量的な成果指標(訪問者数、契約企業数、ワーカー数)
- 測定方法を明記(Google Analytics、CRM等)
- 達成可能性の高い現実的な目標

### 3. 補助事業と販路開拓の関連性を明確に
- Webサイト→検索流入→問い合わせ増加の流れを説明
- 広告→認知度向上→顧客獲得の因果関係を明記
- 展示会→直接対話→信頼構築→商談化の効果を記載

### 4. 補助事業終了後の継続性を示す
- 収益モデル(取引手数料18%)の説明
- 黒字化のタイミング(Month 2-3)
- 自動化による持続可能な運営

### 5. 賃金引上げ要件の確認
- 事業場内最低賃金が地域別最低賃金+50円以上であることを確認
- 賃金台帳に全従業員の時給が明記されていることを確認
- 雇用条件通知書に賃金条件が明記されていることを確認
- 補助事業実施期間終了時にも要件を満たす計画であることを確認
